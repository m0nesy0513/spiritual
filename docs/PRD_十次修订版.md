# 1. 产品背景与定位

## 1.1 背景说明

用户在使用朋友圈、小红书、微博、抖音、B站等社交媒体时，可能被他人的生活展示、人设包装、精致内容、恋爱状态、消费水平、成绩表现、事业进展、身材外貌等内容触发焦虑。这些焦虑不一定来自真实失败，而常常来自选择性展示、背景信息缺失、片段化内容被误认为完整现实，以及平台内容反复强化比较心理。

本产品希望通过“截图分析 + 人设拆解 + 情绪解释 + 心理学方法辅助 + 具体建议 + 百宝箱知识体系”的方式，帮助用户从社交媒体引发的焦虑、不安、比较、自我否定中抽离出来。

## 1.2 产品类型

- 面向社交媒体焦虑的网站工具。

- 第一版优先做网站，手机端优先，电脑端也要支持。

- 第一版是朋友内测完整体验版，不是极简 MVP，也不是粗糙 Demo。

## 1.3 产品一句话定位

一个面向社交媒体焦虑的网站工具。用户上传朋友圈 / 小红书等社交媒体截图后，产品帮助用户解析截图中的人设包装、隐藏信息、比较陷阱与被忽略的细节，并结合用户可选填写的感受，帮助用户从焦虑、不安、自我否定等负面情绪中抽离出来，最后给出解释与建议。

# 2. 核心概念定义

## 2.1 人设存档

“人设存档”不是存档用户自己的人设，而是对社交媒体中容易引发焦虑的人设形象进行识别、记录、拆解和分析。产品不帮助用户打造自己的人设，而是帮助用户看清他人展示出来的人设如何影响自己。

- 精致女生、成熟男生、厉害学生、自律达人、松弛感生活者、完美恋爱人设、高质量生活方式人设、事业成功者、学霸、恋爱赢家、高消费生活方式人设、高情商社交达人、上岸 / 绩点 / 成绩优秀人设、身材 / 外貌优势人设、旅行 / 生活方式人设、熟人朋友圈成功人设等。

### 识别输出规则

- AI 可输出多个相关人设标签。

- 结果页展示全部识别到的人设标签；历史列表展示主标签。

- 当无法可靠判断时，不强行贴标签，显示“当前信息不足，无法明确识别人设类型”。

## 2.2 情绪脱套

“情绪脱套”指用户被社交媒体内容引发焦虑后，产品通过解释、拆解和建议，帮助用户从比较、自我否定和情绪裹挟中抽离出来。

情绪脱套不是简单输出“别焦虑”“别想太多”，而是需要完成：识别触发内容、拆解人设包装、解释情绪来源、区分事实与推测、帮助用户恢复现实感、给出认知调整方法和具体可执行建议。

# 3. 目标用户与核心价值

## 3.1 第一批用户

- 用户本人。

- 用户身边朋友。

- 朋友内测用户。

## 3.2 目标用户范围

所有在使用社交媒体过程中产生焦虑、困惑、不安、比较、自我否定等负面情绪，并且希望获得解释、拆解和帮助的人。目标用户不限制年龄、性别、身份、职业、学历、城市和使用平台。

## 3.3 焦虑类型覆盖

- 外貌 / 身材焦虑

- 精致生活焦虑

- 学习 / 成绩 / 绩点 / 上岸焦虑

- 恋爱 / 被爱 / 亲密关系焦虑

- 消费 / 旅行 / 生活方式焦虑

- 事业 / 实习 / 赚钱 / 成功焦虑

- 熟人朋友圈比较焦虑

- 博主人设压迫感

- 自律人设焦虑

- 松弛感人设焦虑

- 阶层差异焦虑

- 社交能力焦虑

- 其他由社交媒体触发的负面情绪

### 焦虑类型判断规则

- AI 判断焦虑类型时应同时参考截图内容、用户填写的感受和产品知识内容体系。

- 用户填写的感受优先用于理解用户主观体验；截图内容用于判断触发源；二者冲突时应在分析中说明“您的描述与截图信息不完全一致”。

- 可输出多个焦虑类型标签；历史列表展示主类型，详情页展示全部类型。

## 3.4 核心价值排序

1.  帮助用户走出难受的感觉。

2.  缓解社交媒体引发的焦虑。

3.  看穿社交媒体人设包装。

4.  理解自己的情绪来源。

5.  获得具体改善建议。

6.  建立长期抵抗比较焦虑的能力。

7.  未来能看到自己战胜过哪些焦虑，获得成就感。

## 4.1 V0.1 定位

V0.1 是朋友内测完整体验版。它不是极简 MVP，也不是只有一个主流程的半成品。第一版需要形成完整闭环，并具备较完整的页面状态、错误状态、账号权限、管理后台和知识内容体系。

- 管理员后台编辑公共内容。

# 5. 产品整体信息架构与核心流程

## 5.1 底部导航

- 首页

- 功能入口

- 中间相机按钮

- 百宝箱

- 我的

中间相机按钮是最明显、最大的主入口，用于上传截图并开始分析。

## 5.2 首页定位

首页是舒缓情绪内容首页，不是功能堆叠页。首页不应堆放大量功能入口，未来功能由“功能入口页”承接。

## 5.3 用户首次使用流程

1. 用户访问网站。

2. 未登录用户进入登录 / 注册入口页。

3. 注册时同意用户协议、隐私政策，并确认已阅读免责声明。

4. 登录 / 注册成功后展示首次进入引导页。

5. 用户可跳过或完成首次进入引导页。

6. 进入首页。

7. 第一次进入首页后弹出新手教程遮罩。

本流程冻结为登录后展示首次进入引导页。未登录用户只能访问登录、注册、找回密码、用户协议、隐私政策、免责声明等必要页面，不能访问首页、百宝箱、分析功能、历史和个人笔记。

## 5.4 核心截图分析流程

8.  用户点击中间相机按钮。

9.  进入上传截图页。

10. 用户上传单张截图。

11. 系统展示缩略图，用户点击缩略图可进入大图预览。

12. 用户可选填写感受、焦虑强度、截图来源平台，并可选开启参考历史。

13. 用户点击“开始分析”。

14. 进入分析等待页。

15. AI 生成分析结果。

16. 分析成功后自动创建历史记录。

17. 用户查看分块分析结果、来源 / 参考依据、免责声明。

18. 用户进入反馈页并提交反馈。

19. 系统更新该条历史记录。

20. 用户选择回首页、看百宝箱、查看历史。

## 5.5 高风险流程

21. 用户上传截图或填写感受。

22. AI 识别到高风险信号。

23. 页面先显示明显风险提醒。

24. 告知产品不能替代专业医疗、心理咨询、心理治疗或医学诊断。

25. 建议用户寻求医院、专业医生、心理咨询师、当地紧急求助渠道或身边可信任的人帮助。

26. 用户选择是否继续查看普通分析。

## 6.2 登录页

- 字段：手机号或邮箱、密码。

- 操作：登录、跳转注册、忘记密码。

- 状态：账号不存在、密码错误、输入格式错误、登录失败、网络异常。

- 未登录用户默认进入登录 / 注册入口，不进入首页。

## 6.3 注册页

- 字段：手机号或邮箱、验证码、密码、用户名。用户名为必填字段，未填写时提示“用户名不能为空”；用户名最多 20 字，支持中文、英文、数字和下划线，不允许纯空格，不强制唯一。注册验证码按凭证类型发送：手机号使用短信验证码，邮箱使用邮件验证码；验证码有效期 5 分钟，可重发；验证码重发冷却时间为 60 秒，冷却期间按钮禁用并显示倒计时。

- 密码规则：至少 8 位，包含字母和数字。注册场景不存在旧密码，不校验“新旧密码是否相同”。

- 注册页提供两个独立勾选框：1）我已阅读并同意用户协议和隐私政策；2）我已阅读并知悉免责声明。两个勾选框均为必选。

- 状态：账号已存在、验证码错误、验证码过期、验证码发送失败、验证码重发冷却中、用户名不能为空、密码不符合规则、用户协议/隐私政策未勾选、免责声明未确认、注册失败。

- 注册成功后自动登录；新用户跳转首次进入引导页，引导已完成的用户跳转首页。

## 6.4 找回 / 重置密码页

- 仅支持通过已绑定的手机号或邮箱找回：手机号发送短信验证码，邮箱发送邮件验证码；验证码有效期 5 分钟，可重发；验证码重发冷却时间为 60 秒，冷却期间按钮禁用并显示倒计时。

- 若用户只绑定一种凭证且该凭证不可用，第一版不提供人工找回；用户需重新注册或等待后续版本的人工验证机制。

- 本页为一个页面内的两步状态：第一步输入已绑定手机号或邮箱并完成验证码验证；第二步设置新密码。重置密码成功后清除当前找回流程状态，跳转登录页，并提示用户使用新密码重新登录。

## 6.5 首次进入引导页

- Slogan 未确认前展示产品一句话定位短版摘要。

- 跳过后进入首页；跳过只影响引导展示，不影响用户已在注册时确认的协议与免责声明。

- “第一次”按账号维度记录，换设备登录后不重复自动展示。

## 6.8 上传截图页

- 第一版仅支持单张截图。

- 移动端支持从相册选择；设备与浏览器支持时显示拍照上传；电脑端不显示拍照上传选项，仅支持点击选择和拖拽上传。

- 支持 PNG、JPG、JPEG、WEBP；不支持 GIF、视频、PDF。

- 单张图片大小上限为 10MB；超过上限显示大小超限提示。

- 上传完成后页面展示缩略图，点击缩略图进入大图预览。

- 重新上传为覆盖单图模式，新图上传成功后替换旧图。

- 可选填写感受，最多 500 字，支持换行。

- 焦虑强度使用 0-10 整数滑块，0 表示完全不焦虑，10 表示非常焦虑；焦虑强度 ≥ 8 视为“极高”，用于高风险提醒规则中的辅助判断。

- 可选截图来源平台：朋友圈 / 小红书 / 微博 / 抖音 / B站 / 其他。

- 可选开启“参考我的历史摘要，让本次分析更贴近我”。上传页开关默认值读取用户在“我的页面”的参考历史偏好设置；若用户未设置过，则默认关闭。当用户没有历史记录时，该选项显示为禁用状态，并提示“分析过更多内容后，可开启此选项”。

- 用户上传截图或填写感受后离开上传页，应提示“分析尚未开始，离开后将丢失当前内容，是否确认离开？”桌面端和移动端浏览器必须保证点击站内跳转、底部导航、返回按钮时触发确认；浏览器关闭 / 刷新等场景第一版可不保证。

## 6.9 图片预览弹层

- 用于查看大图、确认图片是否清晰、重新上传或关闭预览。

- 系统检测到图片文字识别困难时，应提示用户重新上传或手动补充关键文字；第一版以 OCR 识别置信度低于可用阈值或无法提取有效文字作为自动提示依据。仅在 OCR 置信度低于阈值或无法提取有效文字时，图片预览弹层内显示“手动补充关键文字”文本输入框，最多 200 字；识别正常时隐藏该输入框或置灰提示“识别正常，无需补充”。用户确认后返回上传页，并显示已补充文字。用户也可在预览时自行判断并重新上传。

- 图片预览弹层按使用场景区分功能：上传页预览支持重新上传和手动补充关键文字；历史详情页预览仅支持查看大图和关闭，不支持重新上传。

## 6.10 分析等待页

- 展示明确等待状态。第一版使用前端轮播文案，多条固定文案依次切换，例如“识别截图、理解内容、匹配知识、生成建议”；该轮播文案用于安抚等待，不依赖后端真实进度。分析超时阈值为 60 秒；超过后进入分析失败状态。若 AI 识别到高风险信号，跳转结果页后自动弹出全屏高风险提醒弹层遮挡结果，用户选择后继续查看或返回首页。

- 提供取消分析按钮；取消后不保存分析结果，返回上传页，并保留用户已上传的截图、已填写的感受、焦虑强度和来源平台，仅清除分析等待状态。

- 分析完成后自动跳转结果页。

- 分析超时或失败时进入失败状态，提供重新分析、重新上传、返回首页。

- 分析超时或失败后，已上传截图、感受、焦虑强度、来源平台、手动补充关键文字默认保留，仅清除分析等待状态；用户选择重新上传时再替换图片，选择返回首页时按离开规则处理。

## 6.11 分析结果页

分析结果页按模块分块展示，不做简版 / 完整版切换。所有分析模块都应出现；信息不足时显示“当前信息不足，无法判断”，而不是直接缺失模块。百宝箱推荐模块除外：无匹配内容时显示“暂无相关推荐”或“当前无匹配知识条目”。

1. 一句话摘要（AI 输出时必须控制在 30 字以内，前端不做截断）

2. 关键词

3. 截图内容概括（AI 输出时必须控制在 100 字以内，前端不做截断）

4. 引起焦虑的人设 / 人格化形象类型

5. 焦虑自身类型

6. 包装与隐藏信息拆解

7. 比较陷阱分析

8. 为什么你会焦虑

9. CBT / 心理学方法辅助

10. 具体建议

11. 相关百宝箱推荐

12. 来源 / 参考依据

13. 免责声明（系统渲染模块，非 AI 生成内容）

- 一句话摘要最多 30 字。

- 关键词以标签形式展示，数量允许 1-8 个，不要求点击跳转；若无法提取任何关键词，显示“暂无关键词”。

- 截图内容概括最多 100 字，长内容可折叠。

- 具体建议输出 1-5 条，按可执行性排序；每条建议不超过 100 字；若当前信息不足以生成任何建议，显示“当前信息不足，无法提供具体建议”。

- CBT / 心理学方法以结构化文字卡片展示，不做交互勾选。

- 相关百宝箱推荐优先展示 3 条卡片；若可推荐内容不足 3 条，则展示实际可推荐数量，不强制凑数；点击卡片进入百宝箱详情页。相关百宝箱推荐由系统基于本次分析匹配到的知识条目标签和分类，在 AI 输出后由后端算法排序生成，不作为 AI 直接输出内容。

- 来源 / 参考依据默认折叠，点击后展开。

- 免责声明由系统引用后台维护的全局免责声明内容渲染，默认展开展示，不可折叠，属于结果页系统渲染模块，不依赖 AI 动态生成；管理员可维护文案，但“不替代医疗/心理咨询/心理治疗/诊断”等核心条款不可删除。

- 模块默认状态：一句话摘要、关键词、截图内容概括、人设类型、焦虑类型、包装拆解、比较陷阱分析、为什么焦虑、CBT 方法、具体建议、百宝箱推荐默认展开；来源 / 参考依据默认折叠；免责声明默认展开且不可折叠。

### 来源 / 参考依据与相关推荐的边界

- 相关百宝箱推荐：面向用户的延伸阅读。

- 来源 / 参考依据：本次分析实际引用或参考的知识条目，用于解释分析依据。

- 来源展示内容应包括知识条目标题、分类、简短摘要、为什么用于本次分析。

### 结果页用户操作

结果页底部操作区固定承接本页操作：保存原截图复选框、保存完整识别文字复选框、重新分析按钮、进入反馈页主按钮、分享 / 收藏 / 导出 / 下载占位入口。保存原截图和保存完整识别文字两个复选框放在底部操作按钮上方，默认均不勾选。备注入口放在结果页“我的备注”折叠卡片内，用户点击后展开文本框填写备注。

复制 AI 输出的纯文本格式为“模块标题：内容”形式，各模块之间用两个换行分隔，不包含 HTML 或 Markdown 标签。示例：一句话摘要：xxx  关键词：xxx，xxx  截图内容概括：xxx。

- 复制 AI 输出。分析结果页复制仅包含 AI 生成模块，格式为带模块标题的纯文本；历史详情页可提供“复制 AI 分析”和“复制完整记录（含用户备注）”两个选项。

- 给 AI 输出添加备注。备注与 AI 结论分区展示，不覆盖、不替换 AI 原文。

- 重新分析。仅用户从上传页完成首次分析后进入的结果页，才适用“未提交反馈则覆盖当前记录”的规则；从历史详情页或历史记录恢复进入结果页时，重新分析统一显示为“基于这条记录重新分析”，并按历史详情页重新分析规则生成新历史记录，不覆盖旧记录。已提交反馈的记录再次打开结果页时，该操作同样显示为“基于这条记录重新分析”。

- 进入反馈页。

- 点击相关百宝箱推荐卡片查看详情。

- 选择是否保存原截图。

- 选择是否保存完整识别文字。

- 结果页重新分析覆盖当前结果时，更新当前历史记录的分析结果内容，保留原历史记录 ID 和分析时间；该操作不删除重建历史记录，同时保留用户此前对“是否保存原截图”和“是否保存完整识别文字”的选择，不重置为默认值。默认情况下不保存原截图和完整识别文字，用户需主动勾选保存。

- 分析成功并创建历史记录后，结果页通过当前历史记录 ID 从历史数据恢复；结果页 URL 包含历史记录 ID，仅对当前登录用户有效，不生成公开分享链接。用户刷新结果页时从历史记录恢复，不重新触发分析。用户访问非本人历史记录 ID 时，系统返回 403 或重定向至首页，不展示任何历史内容；该权限校验必须同时在前端路由和后端数据/API 层执行。

## 6.12 高风险提醒状态

- 出现明确自伤、自杀、伤害自己、不想活、想消失、极端绝望等表达时触发。高风险提醒采用三态规则：未触发、待处理、已处理。首次分析完成后若识别为高风险，历史记录写入高风险标记并将提醒状态设为“待处理”；结果页弹出全屏提醒。若用户首次未处理提醒（关闭浏览器或异常退出），再次从历史详情页打开该记录时仍触发一次；用户点击“继续查看”或“返回首页”后，提醒状态改为“已处理”，后续再次打开不重复弹出。

- 用户填写焦虑强度极高（0-10 量表中 ≥ 8），并同时出现明显危险表达时触发。

- 高风险提醒采用全屏弹层。系统先完成 AI 分析、生成结果并创建历史记录，再弹出高风险提醒遮挡结果页；用户选择“继续查看”后移除弹层，直接展示已生成的普通分析结果。用户选择“返回首页”后，分析结果和历史记录仍已保存，用户可在历史记录中查看，首页无需强制提示未查看记录。

- 用户选择“返回首页”后，已生成的分析结果和历史记录仍然保留，用户可在历史记录中查看；首页不强制提示未查看记录。

- 语义风险示例：表达生命无意义、想离开这个世界、想永远消失、无法承受痛苦并伴随行动暗示、明显表达自我伤害倾向等。暗示行动包括：表达想实施自伤的具体方式、表达已准备工具或计划、表达“今晚/现在/马上就想结束”等时间与行动绑定表述。AI 判断不确定但风险较高时，优先触发提醒。

- 本产品不提供危机干预，不替代专业帮助。

## 6.13 反馈页

- “现在有没有好一点”：好很多 / 好一点 / 没有变化 / 更糟了。

- “这次分析有没有用”：很有用 / 有点用 / 一般 / 没用。

- 分析后焦虑强度：0-10 整数滑块，非必填。

- 后续感受反馈：可选文本，最多 500 字；提交前可编辑，提交后不可直接修改，后续补充通过历史详情页备注完成。

- 若用户填写了分析前焦虑强度，反馈页以数字 + 横向进度条展示前后强度对比，例如“分析前：7/10 → 分析后：4/10”。

- 反馈为强烈建议但非强制步骤。反馈完成状态为反馈页内嵌状态，不作为独立页面；提交成功后停留在完成状态，提供回首页 / 看百宝箱 / 查看历史三个按钮，不自动跳转；其中“查看历史”跳转至当前分析记录的历史详情页。用户也可跳过反馈，跳过不影响历史保存；第一版不允许在历史详情页补充反馈，跳过视为放弃本次反馈。

- 提交失败时保留用户已填写内容，提示失败并提供重试。

## 6.14 历史记录页

- 默认视图为按时间排序列表，最新在前。

- 提供“按人设类型分组查看”切换项；切换状态第一版不要求记忆。

- 分组查看时只展示有历史记录的人设类型分组，不展示空分组。

- 每条历史展示：分析时间、截图缩略图或占位图、主人人设标签、主焦虑类型、一句话摘要、反馈结果摘要。反馈结果摘要以标签展示“有没有好一点”和“有没有用”的选项；未提交反馈时显示“未反馈”。

- 未保存原截图时展示统一占位图标，保持列表高度一致。

- 时间筛选支持：今天、本周、本月、自定义时间范围；自定义范围使用起始日期-结束日期双日期选择器，最长跨度不超过 1 年。

历史页顶部提供筛选栏和搜索入口：筛选栏包含时间、焦虑类型、人设类型、标签筛选；右侧提供搜索图标或搜索输入框。焦虑类型和人设类型使用下拉选择器，标签筛选使用多选标签选择器，与“按人设类型分组查看”切换项并列展示。

- 筛选和搜索可同时使用，逻辑为交集。

历史自定义标签采用逐个添加方式（输入后回车或点击添加），最多 5 个，单个标签最多 10 字。

## 6.15 历史详情页

- 展示完整 AI 分析结果、用户感受（只读）、截图来源平台、焦虑强度、反馈结果、后续感受反馈（只读）、用户备注、用户自定义标签、分析时间、原截图和完整识别文字（若用户选择保存）。未填写用户感受时显示“未记录感受”；若仅填写分析前强度，历史详情页只展示分析前强度，不展示对比；若仅填写分析后强度，历史详情页只展示分析后强度，不展示对比；若填写前后强度，展示前后对比；若两者都未填写，隐藏强度区域；未提交反馈时显示“未反馈”。历史详情页中的来源/参考依据和免责声明均以分析时快照保存，不受后续知识库停用、编辑或免责声明文案更新影响。未保存原截图时，截图区域显示“未保存截图”的占位文字或与列表一致的占位图标；若保存原截图，用户点击原截图后进入图片预览弹层查看大图。

- 历史详情页顶部或底部操作区提供“复制 AI 分析”和“复制完整记录（含用户备注）”两个按钮；“复制 AI 分析”只复制 AI 生成模块，“复制完整记录”复制 AI 分析、用户备注、标签和反馈摘要。历史详情页同样展示分享 / 收藏 / 导出 / 下载占位入口，第一版为不可用或 Coming Soon 状态，不生成真实公开链接或导出文件。备注和标签编辑入口位于“我的备注 / 我的标签”独立区域，不混入 AI 结论区。

- 完整识别文字默认折叠，点击展开查看。OCR 完整识别文字与用户手动补充关键文字分开展示；若用户未保存完整识别文字，但曾手动补充关键文字，则仍展示该补充文字。第一版完整识别文字与截图独立展示，不做截图文字高亮或区域标注。

- 用户可添加、修改、删除备注和标签。备注最多 500 字，支持换行，不支持富文本。

- 删除历史时，删除该历史相关的截图、识别文字、感受、分析结果、反馈、备注、标签关联；不删除关联的个人笔记，只解除关联。

- 历史详情页可发起“基于这条记录重新分析”，该操作生成新的历史记录，不覆盖旧记录。重新分析默认复用原截图、原感受、原焦虑强度、原截图来源平台、原参考历史设置，仅重新触发 AI 分析；默认不保存原截图和完整识别文字，用户可在重新分析前重新勾选并修改这些输入。

## 6.16 百宝箱页与详情页

- 百宝箱是知识集合区，类似百科全书 / 词典。

- 百宝箱页结构：推荐内容、搜索框、分类浏览、内容列表、标签、空状态。百宝箱搜索框常驻位于推荐内容下方、分类标签栏上方；用户输入关键词后在当前百宝箱页内展示搜索结果。内容列表默认按更新时间倒序排列，最新更新内容在前。

- 默认展示“全部”分类；分类导航采用横向标签栏。

- 分类采用单层结构；每条内容属于一个主分类，可附加多个标签。

- 详情页展示标题、分类、标签、正文、相关内容推荐。

- 正文第一版统一使用标题 + 正文 + 标签的标准详情页，不做复杂交互。

- 百宝箱详情页相关内容推荐优先展示 3 条；推荐逻辑：优先同分类且同标签，其次同分类，最后按更新时间取同分类其他内容；仍不足时展示其他核心分类的已启用内容；若全站可推荐内容不足 3 条，展示实际可推荐数量，不强制凑数。百宝箱首页顶部为管理员手动推荐的 3-5 条置顶卡片；若无手动推荐，顶部推荐区按更新时间倒序自动展示前 5 条。推荐区下方仍展示按更新时间倒序排列的内容列表；前端渲染时，内容列表中已置顶内容不再重复展示；数据库查询不做排除，以支持后台完整管理和分页计算。

- 点击分类返回该分类列表；点击标签进入百宝箱该标签筛选结果。点击标签后进入跨分类标签筛选视图，分类导航栏保持显示“全部”。

## 6.17 我的页面

我的页面是个人中心页面，集中承接账号信息、个人设置、帮助说明、历史入口、个人笔记快捷入口、用户提议 / 反馈入口和管理员后台入口。

页面展示：头像、用户名、注册时间、已绑定手机号 / 邮箱状态。

资料编辑：提供修改用户名和修改头像入口。用户名为必填字段，最多 20 字，支持中文、英文、数字和下划线，不允许纯空格，不强制唯一；头像支持 JPG / PNG，大小上限为 2MB。头像超过 2MB 时提示“头像大小超过 2MB，请重新上传”，不自动压缩；未上传时使用系统根据用户名首字母生成的彩色默认头像。

账号操作：修改密码、补充绑定第二个凭证、退出登录、注销账号。

修改密码以独立页面或弹层状态承接，字段包含旧密码、新密码、确认新密码；新密码至少 8 位，包含字母和数字，且不能与旧密码相同；错误状态包括旧密码错误、新密码不符合规则、两次输入不一致、修改失败。

补充绑定第二个凭证以独立页面或弹层状态承接，字段包含新手机号或新邮箱、验证码；系统向新凭证发送验证码，验证通过后绑定；错误状态包括验证码错误、验证码过期、凭证已被绑定、发送失败。

退出登录使用确认弹层，按钮为“取消”和“确认退出”；注销账号使用二次确认页或弹层，要求用户明确确认注销后再执行，提示该操作将物理删除私人数据且不可恢复。

功能入口：历史记录、个人笔记、产品说明、免责声明、新手教程、用户提议 / 反馈。

入口归属说明：历史记录、产品说明、免责声明、新手教程只在“我的页面”提供入口，不在功能入口页重复出现；个人笔记和用户提议 / 反馈同时在“我的页面”和“功能入口页”提供入口。

偏好设置：参考历史默认设置，默认关闭，仅影响未来新发起的分析，不触发历史记录重新分析。

占位入口：长期复盘、成就墙、焦虑档案。

管理员后台入口仅管理员账号可见，普通用户不可见；同时必须配置路由守卫和 API 权限校验，非管理员账号直接访问管理员路由或调用管理员 API 时返回 403。

状态：资料加载失败、修改资料失败、修改密码失败、绑定凭证失败、退出登录确认、注销账号二次确认。

## 6.18 个人笔记页

- 入口放在功能入口页；我的页面包含个人笔记入口作为快捷入口。

- 支持新增、编辑、删除、搜索、标签、关联历史分析、关联百宝箱内容。个人笔记页顶部常驻搜索框，搜索范围为笔记标题、正文和标签。

- 个人笔记标签与历史记录自定义标签为两套独立标签系统，第一版不共享标签库。

笔记标签采用逐个添加方式（输入后回车或点击添加），最多 5 个，单个标签最多 10 字。

- 笔记标题最多 60 字，正文最多 5000 字。

- 笔记编辑页提供“关联来源”搜索选择器，可搜索历史记录的一句话摘要和百宝箱标题；关联关系存储为目标 ID + 目标标题快照 + 目标类型。关联后显示来源标题，点击可跳转详情页。关联历史记录被删除后显示“关联来源已删除”；关联百宝箱内容被管理员停用后，笔记中显示原标题并标记为“内容已停用”，第一版不再跳转详情页。

- 空状态展示“还没有笔记，写下第一条反焦虑笔记”并提供新增按钮。

## 6.19 功能入口页与 Coming Soon

- 功能入口页保留底部导航。

- 功能入口页共 6 个主要卡片：个人笔记（可用）、用户提议 / 反馈（可用）、情绪疏导室（Coming Soon）、长期复盘（占位）、成就墙（占位）、焦虑档案（占位）。不再单独列“敬请期待占位卡”；如需新增未来功能，占位卡数量另行确认。

- Coming Soon 页面需要好看、有简短说明、有返回按钮、有回首页按钮，视觉风格统一。

## 6.20 用户提议 / 反馈提交页

- 入口在“我的页面”和功能入口页中均包含，二者进入同一个提交页。

- 提议类型：功能建议、Bug 反馈、内容纠错、体验优化、其他。

- 第一版仅支持文字提交，不支持附件上传。

- 联系方式可选，为自由文本输入，最多 100 字；表单提示“建议填写手机号或邮箱，方便我们联系您”。若用户填写，管理员可以在后台看到该联系方式，用于联系用户。

- 第一版用户提交后无法查看管理员处理状态，管理员处理结果不反向通知用户；后续可扩展“我的反馈”入口。

- 提交成功显示“反馈已提交”；提交失败保留输入并提供重试。

- 管理员处理状态列表固定为：未处理、处理中、已采纳、不采纳、已完成。

## 6.21 管理员后台页

- 管理员账号由项目创建者预先指定，普通用户不能自助申请管理员；第一版通过部署配置或初始化数据指定管理员账号，不采用“第一个注册账号自动成为管理员”的方式。更换管理员账号需要由项目创建者修改配置或初始化数据，不提供普通用户界面自助变更。

- 管理员后台结构包含以下导航和功能区：百宝箱管理、首页内容管理、用户反馈 / 提议、系统文案设置；右侧内容区以列表 / 表格展示，点击后进入编辑页或编辑弹层。

- 管理员入口仅管理员账号可见。

- 管理员可编辑公共内容：百宝箱、百宝箱首页推荐、首页名言、好歌推荐、轻柔情绪问候、功能入口占位内容、Coming Soon 文案、产品说明、免责声明、新手教程、用户提议 / 反馈状态。用户提议 / 反馈处理状态固定为：未处理、处理中、已采纳、不采纳、已完成。

- 管理员不能查看用户历史、截图、感受、分析结果、反馈记录、后续感受、个人笔记、标签、备注。

- 百宝箱管理采用列表 / 表格展示，支持新建、编辑、启用 / 停用；编辑页 / 弹层字段至少包含标题、分类、标签、正文、适用场景、来源说明、启用状态、推荐 / 置顶到百宝箱首页开关；表单校验沿用 9.2.1 规则。百宝箱首页推荐通过条目上的“推荐 / 置顶到百宝箱首页”开关控制，管理员可选择 3-5 条已启用内容作为百宝箱首页推荐。

- 用户反馈 / 提议管理采用列表展示，支持按状态筛选，并可通过下拉或编辑操作修改处理状态。系统文案设置包含：免责声明、产品说明、新手教程内容、轻柔情绪问候；编辑页 / 弹层字段包含文案类型、正文内容、保存按钮、取消按钮和预览区域。第一版不记录管理员操作日志，后续版本可扩展。

- 管理员编辑公共内容后第一版可立即生效；百宝箱内容应支持启用 / 停用状态。

# 8. AI 功能边界

## 8.1 AI 定位

AI 是第一版核心能力之一，负责围绕社交媒体截图和用户情绪完成结构化拆解与辅助。AI 不是普通聊天，不提供医疗诊断或心理治疗。

## 8.2 AI 输入

- 用户上传的截图原图。第一版 AI 优先基于多模态能力直接分析截图图像。

- OCR 识别文字或用户手动补充的关键文字。OCR 文字作为辅助输入，用于截图文字提取、关键词匹配和知识库检索兜底；当 OCR 失败时，允许用户手动补充关键文字。

- 用户可选填写的感受。

- 分析前焦虑强度。

- 截图来源平台。

- 产品知识内容体系中与本次内容相关的已启用知识条目。

- 用户主动开启时的历史摘要。

## 8.3 AI 输出

- 一句话摘要，最多 30 字。

- 关键词，允许 1-8 个；关键词由 AI 基于截图内容、用户感受和分析结果生成，不强制与 OCR 提取文字完全一致。

- 截图内容概括，最多 100 字。

- 人设 / 人格化形象类型。

- 焦虑类型。

- 包装与隐藏信息拆解。

- 比较陷阱分析。

- 为什么用户会焦虑。

- CBT / 心理学方法辅助。

- 1-5 条具体建议，每条不超过 100 字。

- 相关百宝箱推荐为结果页系统推荐模块，由系统基于本次分析匹配到的知识条目标签和分类排序生成，不作为 AI 直接输出内容。

- 来源 / 参考依据。

- 说明：免责声明为结果页系统渲染模块，非 AI 生成内容，不属于 AI 输出。

## 8.4 AI 输出依据

AI 必须优先基于产品知识内容体系输出，不应完全依赖大模型自由发挥。每次分析应基于截图 OCR 关键词、用户感受关键词、识别到的人设类型和焦虑类型，从知识库中匹配分类和标签相关的已启用条目，优先选择“适用场景”与本次分析最匹配的条目，每次注入 5-8 条相关知识用于分析，并在结果页展示参考依据；匹配结果不足 5 条时注入全部匹配到的已启用条目，不强制凑数；匹配结果为 0 条时，AI 基于通用知识生成分析，并在来源 / 参考依据中标注“本次分析未匹配到特定知识条目”。

本文不规定具体技术实现方式。后续技术文档可以再决定检索、调用、缓存、上下文组织等方案。

## 8.5 AI 参考历史规则

- 默认不参考历史。

- 用户可在上传页临时开启，也可在我的页面设置默认偏好。

- 开启后只参考历史摘要，不看完整历史详情、原截图、完整个人笔记。

- 可参考：历史一句话摘要、人设类型、焦虑类型、关键词、反馈概况。

- 历史摘要在每次分析完成后自动生成并保存，用于后续分析；字段结构包含一句话摘要、人设类型、焦虑类型、关键词、反馈概况。反馈概况由“有没有好一点”选项 + “有没有用”选项组成，例如“好很多，很有用”；未提交反馈时为空。 用户开启参考历史时，历史摘要以结构化文本形式注入 AI 分析上下文，作为用户背景信息参考，不直接替代本次分析的知识库匹配。

## 8.6 重新分析规则

- 仅用户从上传页完成首次分析后进入的结果页，才适用“未提交反馈则覆盖当前结果”的规则；覆盖时更新当前历史记录的分析结果内容，保留原历史记录 ID 和分析时间，同时保留用户此前对“是否保存原截图”和“是否保存完整识别文字”的选择，不重置。从历史详情页或历史记录恢复进入结果页时，重新分析统一显示为“基于这条记录重新分析”，生成新的历史记录；已提交反馈后的记录再次打开结果页时同样生成新历史记录，不覆盖旧记录。

- 从历史详情页重新分析，生成新的历史记录，不覆盖旧记录；默认复用原截图、原感受、原焦虑强度、原截图来源平台、原参考历史设置；默认不保存原截图和完整识别文字，用户可在重新分析前重新勾选并修改这些输入。

- 重新分析可能得到相似结果，页面可提示“重新分析会从同一材料出发，结果可能相似”。

- 第一版不限制次数；后续可根据资源情况再考虑限制。

## 8.7 高风险识别

- 明确表达自伤、自杀、伤害自己、不想活、想消失等内容时触发。

- 表达极端痛苦、绝望，并伴随危险暗示时触发；用户焦虑强度 ≥ 8 且同时出现明显危险表达时，也应触发高风险提醒。

- AI 不进行医学判断，只做风险提醒。

- 误报优先于漏报，用户仍可选择继续查看普通分析。对“活着没意思”“想离开这个世界”“想永远消失”“无法承受并暗示行动”等语义信号，AI 判断达到较高风险倾向时应触发提醒；不要求给出医学风险等级。“暗示行动”包括表达想实施自伤的具体方式、表达已准备工具或计划、表达带时间点的行动倾向（如“今晚就想结束”）等。高风险提醒出现前，系统已完成本次分析并创建历史记录；弹层仅遮挡结果，不删除结果。高风险标记保存在历史记录中，随历史记录删除而删除。

## 8.8 AI 输出语气

状态：待确认。候选方向包括温柔安抚型、专业清醒型、温柔 + 清醒结合、有一点看穿社交媒体套路的爽感、分析型但不冷漠、安慰型但不说教。

# 9. 知识内容体系与百宝箱

## 9.1 知识体系定位

知识内容体系是 AI 分析、百宝箱内容、来源 / 参考依据和内测质量评估的共同支撑。百宝箱是知识体系面向用户的前端展示视图。

## 9.2 知识条目字段

| **字段** | **说明**                                                                                   |
|----------|--------------------------------------------------------------------------------------------|
| 标题     | 知识条目名称。                                                                             |
| 分类     | 单一主分类，例如焦虑类型、人设图鉴、CBT 方法等。                                           |
| 标签     | 可多个，用于搜索和关联；最多 5 个，单个标签 10 字以内。                                    |
| 正文     | 用户可阅读的内容。第一版正文支持 Markdown 子集：标题、列表、加粗、链接；不支持图片和视频。 |
| 适用场景 | 说明该知识适合用于哪些分析场景。                                                           |
| 来源说明 | 可记录内容依据或备注，不要求第一版接入外部来源。                                           |
| 启用状态 | 启用后可展示并可被 AI 参考；停用后不展示。                                                 |
| 更新时间 | 用于管理员维护。                                                                           |

### 9.2.1 管理员新建 / 编辑知识条目的表单规则

- 标题必填，最多 60 字。

- 分类必填，单选；标签可选，最多 5 个，单个标签最多 10 字。

- 正文必填，最多 5000 字，支持 Markdown 子集（标题、列表、加粗、链接），不支持图片和视频。

- 适用场景可选，最多 200 字。来源说明可选，最多 200 字；更新时间自动记录，不允许管理员手动修改。

- 启用状态必填；未启用内容不展示、不被 AI 引用。

## 9.3 第一版最小知识内容包

| **分类**              | **最少条数** | **说明**                                                 |
|-----------------------|--------------|----------------------------------------------------------|
| 焦虑类型解释          | 12           | 覆盖常见社交媒体焦虑。                                   |
| 人设图鉴              | 12           | 覆盖常见引发比较的人设。                                 |
| 社交媒体套路拆解      | 8            | 选择性展示、滤镜、叙事包装等。                           |
| CBT / 心理学方法      | 8            | 事实-推测区分、自动化想法、认知重评等。                  |
| 情绪急救步骤          | 6            | 当下缓解方法。                                           |
| 行动建议库            | 6            | 可执行建议。                                             |
| 自我安抚 / 反比较训练 | 5            | 非医疗性质的自我帮助内容。                               |
| 典型案例              | 3            | 第一版使用虚构或明确标注的示例案例，不使用真实用户数据。 |

进入完整内测的最低知识库条件为：第一版总计不少于 60 条，且每个核心分类至少 3 条已启用内容，且不得出现空标题或空正文。若内容量不足，知识库可开发但不应进入完整内测验收。

## 9.4 来源展示规则

- 来源 / 参考依据只引用产品知识体系中的已启用条目。

- 来源默认折叠，用户点击展开查看。

- 来源需要说明引用了什么，以及为什么与本次分析有关。“为什么与本次分析有关”由 AI 基于知识条目的适用场景和本次分析关键词匹配关系生成，最多 50 字，固定格式为：“因为本次分析涉及 \[关键词\]，与 \[知识条目\] 的 \[适用场景\] 相关。”当匹配知识条目为 0 条时，来源区使用特殊提示：“本次分析未匹配到特定知识条目，基于通用知识生成。”

历史记录中的来源 / 参考依据以快照形式保存，保存分析时引用的知识条目标题、分类、简短摘要与关联说明；后续知识条目被停用或编辑，不影响已有历史详情中的来源展示。百宝箱详情页实时查询时只展示已启用内容。来源 / 参考依据默认展示全部注入的已启用知识条目；不足 5 条时展示全部匹配项；0 条时展示上述特殊提示。

- 相关百宝箱推荐用于延伸阅读，来源 / 参考依据用于解释本次分析依据，两者不得混淆。

# 10. 数据与内容规则

## 10.1 自动保存时机

- AI 分析成功生成结果时，自动创建历史记录。

- 用户提交反馈后，反馈内容更新到该历史记录。

- 如果用户未提交反馈就离开，历史记录仍保留，反馈字段为空。

## 10.2 自动保存内容

- 用户感受、分析前焦虑强度、截图来源平台、手动补充关键文字（可选，最多 200 字，与完整识别文字分开保存）。

- 完整结构化分析结果。

- 一句话摘要、关键词、人设类型、焦虑类型。

- 历史摘要，结构包含：一句话摘要、人设类型、焦虑类型、关键词、反馈概况。

- 分析时间。

- 反馈结果，若用户提交。

- 来源 / 参考依据快照，包含知识条目标题、分类、简短摘要、关联说明；免责声明快照，保存分析时全局免责声明文本。

- 高风险标记与提醒处理状态，作为历史记录字段保存；字段至少包含是否高风险、提醒状态（待处理 / 已处理）和触发依据摘要。

## 10.3 用户自选保存内容

- 原截图。

- 完整识别文字。

用户在结果页选择是否保存原截图和完整识别文字，默认状态为不保存，用户需主动勾选保存。该选择仅影响历史记录附件存储，不影响 AI 分析本身。第一版不提供单独撤销保存原截图的功能，原因是简化数据管理逻辑、避免附件与历史记录状态不一致；如需彻底删除原截图，可删除整条历史记录，或注销账号删除全部私人数据。

## 10.4 可编辑与不可编辑

| **类型**     | **内容**                                                                                                                                                                                                                               |
|--------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 用户可编辑   | 用户感受仅在分析前（上传页）可编辑；生成分析结果后不可修改，若需补充说明通过备注完成。用户可编辑内容包括：自定义标签、备注、头像、用户名、密码、反馈结果（提交前）、个人笔记；后续感受反馈仅在反馈页提交前可编辑，提交后不可直接修改。 |
| 用户不可编辑 | AI 生成分析、人设识别、焦虑类型、包装拆解、CBT 调整、建议、摘要、关键词、百宝箱推荐。                                                                                                                                                  |
| 备注边界     | 备注是用户自己的想法记录，不能覆盖或替换 AI 原文。UI 上应分为 AI 结论区和用户备注区。                                                                                                                                                  |

## 10.5 删除规则

- 删除历史：采用物理删除，从数据库删除该历史记录及其截图、识别文字、用户感受、分析结果、反馈、后续感受、备注、标签关联、来源 / 参考依据快照、免责声明快照和高风险标记。

- 删除历史不删除个人笔记；如笔记关联该历史，则解除关联并显示“关联来源已删除”。

- 删除个人笔记：仅删除该笔记，不影响历史或百宝箱内容。

- 注销账号：采用物理删除用户私人数据，包括历史记录、截图、识别文字、感受、分析结果、反馈、后续感受、备注、个人笔记、标签、头像、用户名、账号凭证和注册时间等账号关联数据。用户主动提交的提议 / 反馈第一版注销后不保留反馈正文，仅匿名化保留提议类型、处理状态、提交时间，并删除用户 ID、用户名、联系方式；后台显示为“已注销用户提交”。

## 10.6 时间与同步

- 所有用户数据应随账号在多设备间同步。第一版采用登录时或进入相关页面时拉取账号数据的方式，不实现多设备实时同步。

- 分析时间以用户点击“开始分析”的时间为准。

- 页面展示时间使用用户本地时间，精度显示到分钟。

# 11. 账号、隐私与权限

## 11.1 登录规则

- 第一版必须登录后才能使用全部功能。

- 未登录用户只能访问登录、注册、找回密码、用户协议、隐私政策、免责声明等必要页面。

- 未登录用户不能访问首页、百宝箱、分析、历史、笔记、我的页面。

## 11.2 账号凭证

- 手机号 / 邮箱作为登录凭证。

- 用户可以先绑定手机号或邮箱其中一个，后续可补充绑定另一个；补充绑定时必须向第二个凭证发送验证码，验证通过后方可绑定。

- 两个已绑定凭证均可用于登录；第一版不支持解绑主凭证或更换已绑定凭证。

- 用户名可以重复，仅作为展示名。

## 11.3 用户协议、隐私政策与免责声明

- 注册时必须同意用户协议和隐私政策。

- 注册时必须确认已阅读免责声明。

- 免责声明为全局内容，结果页每次显示。

- 免责声明核心条款不可被管理员删除：不替代专业医疗、不替代心理咨询、不替代心理治疗、不提供医学诊断、严重风险时寻求专业帮助。

## 11.4 管理员边界

- 管理员账号由项目创建者预先指定。

- 管理员只能管理公共内容和用户主动提交的提议 / 反馈。

- 管理员不能查看用户私人历史、截图、感受、分析结果、反馈记录、后续感受、个人笔记、标签、备注。

- 用户主动填写的联系方式可在提议 / 反馈后台中对管理员可见，表单需提前说明。

## 11.5 内测使用情况记录

- 产品内可记录匿名或汇总的使用情况，用于判断流程是否可用。

- 如需要记录具体朋友编号、具体使用行为，应作为内测反馈表的人工记录或在用户知情情况下记录。

- 不得通过管理员后台暴露用户私人内容来做内测分析。

# 12. 搜索需求

## 12.1 搜索类型

- 首页统一搜索：搜索历史记录、百宝箱内容、个人笔记。

- 历史局部搜索：仅搜索历史。

- 百宝箱局部搜索：仅搜索百宝箱。

- 个人笔记局部搜索：仅搜索笔记。

## 12.2 搜索规则

- 首页统一搜索在首页顶部展开结果面板，按历史记录、百宝箱内容、个人笔记分组展示。每组默认展示前 5 条结果，支持点击“查看更多”进入对应模块的局部搜索结果视图；点击“查看更多”时携带当前关键词进入对应模块，自动填充局部搜索框并自动执行搜索，跳转进入时默认清空该模块原有筛选条件。若用户之后手动设置筛选，则搜索关键词与筛选条件按交集生效。

- 每组内部优先按相关度排序；历史记录可兼顾时间新旧。

- 历史筛选以历史页顶部筛选栏展示；时间、焦虑类型、人设类型、标签筛选可与搜索同时使用。历史页标签筛选指用户为历史记录添加的自定义标签，不包含 AI 关键词和百宝箱标签。

- 百宝箱分类以横向滚动标签栏展示；默认展示“全部”。

- 局部搜索只搜索当前模块，不触发全局搜索；第一版搜索触发方式支持点击搜索按钮或回车。

- 搜索匹配字段：历史搜索匹配一句话摘要、人设类型、焦虑类型、关键词、用户自定义标签；百宝箱搜索匹配标题、正文、标签；个人笔记搜索匹配标题、正文、标签。

- 搜索可与筛选同时使用，逻辑为交集。

- 第一版不要求输入即实时搜索。

# 14. 验收标准

## 14.1 总体验收

- 核心流程可完整跑通。

- 核心页面在手机端主流机型上无关键按钮遮挡、无无法点击区域、无阻断流程的布局错位。

- 每个已确认页面均有至少 1 个可点击的返回、退出或下一步操作；每个已确认入口均能进入对应页面或状态；每个已确认状态均有明确文案和下一步按钮，不出现死路页面。

- 失败、空状态、异常状态都有明确提示和下一步按钮。

- 用户数据隔离无串号。

- 管理员后台可用且不越权。

- 所有核心页面均有明确标题、主要操作按钮、返回路径、加载 / 成功 / 失败状态；不得出现空白页、死路页面或未替换的临时占位文本。

## 14.2 可量化内测指标

- 手机端测试范围至少覆盖 1 款 iOS 主流机型和 1 款 Android 主流机型；核心流程页面元素不得出现无法点击、遮挡关键按钮、阻断流程的布局问题。

- AI 有用性 = 提交反馈用户中选择“有点用”或“很有用”的人数 / 提交反馈总人数，目标为 ≥70%。

核心流程完成率定义为：≥90% 的内测用户能完成一次截图分析（从上传截图到查看分析结果）。反馈提交率另行统计，不作为核心流程完成率的必要条件。

复用意愿 = 填写内测问卷用户中选择“会”或“可能会”的人数 / 填写问卷总人数，目标为 ≥70%；问卷问题为“你以后刷到类似内容，还会不会想打开这个产品？”。

| **指标**       | 目标                                                                                                                                                               |
|----------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 核心流程完成率 | ≥ 90% 的内测用户能完成一次截图分析（从上传截图到查看分析结果）；反馈提交率另行统计。                                                                               |
| 严重问题       | P0 必须为 0；P1 必须处理完。                                                                                                                                       |
| 手机端可用性   | 核心流程测试页面中，关键按钮不可点击、关键内容被遮挡、主要布局错位等阻断问题为 0；轻微错位作为 P2 记录。                                                           |
| 复用意愿       | 填写内测问卷用户中选择“会”或“可能会”的人数 / 填写问卷总人数 ≥ 70%。                                                                                                |
| AI 有用性      | 提交反馈用户中选择“有点用”或“很有用”的人数 / 提交反馈总人数 ≥ 70%。                                                                                                |
| 知识库支撑     | 有匹配知识条目时，来源 / 参考依据必须对应分析时已启用的有效知识条目；匹配结果为 0 条时，必须展示“本次分析未匹配到特定知识条目，基于通用知识生成”，且不得伪造来源。 |

## 14.3 分析结果完整性验收

一次完整分析结果页必须展示 13 个模块：11 个 AI 生成模块（一句话摘要、关键词、截图内容概括、人设类型、焦虑类型、包装与隐藏信息拆解、比较陷阱分析、为什么会焦虑、CBT / 心理学方法辅助、具体建议、来源 / 参考依据）+ 2 个系统渲染模块（相关百宝箱推荐、免责声明）。AI 生成模块信息不足时用“无法判断”占位说明，不能直接缺失；百宝箱推荐无匹配内容时显示“暂无相关推荐”；免责声明由后台全局内容渲染，不由 AI 生成。

## 14.4 账号与权限验收

- 能注册、登录、退出、找回密码、修改密码、注销账号。

- 注册时协议与免责声明确认完整。

- 用户 A 不能看到用户 B 的任何私人内容。

- 管理员不能查看用户私人内容。

- 管理员能处理公共内容和用户主动提交的提议 / 反馈。

## 14.5 知识库验收

- 第一版知识内容不少于 60 条。

- 完整内测验收最低条件：总计不少于 60 条，且每个核心分类至少配置 3 条已启用内容，并且无空标题、无空正文。

- 有匹配知识条目时，AI 结果中的来源 / 参考依据必须对应分析时已启用的有效知识条目；匹配结果为 0 条时，必须展示“本次分析未匹配到特定知识条目，基于通用知识生成”，且不得伪造来源。

- 百宝箱可搜索、分类、查看详情。

- 管理员可启用 / 停用 / 编辑知识条目。

# 附录 B. 文档结论

本 PRD 修订版保留原产品目标：帮助用户从社交媒体焦虑中抽离出来。修订重点不是缩小产品，而是把原 PRD 中容易导致开发误解的冲突、缺页、边界和验收问题补齐。

第一版核心闭环为：登录 / 注册 → 首次引导 → 首页 → 上传截图 → AI 拆解 → 结果解释 → 风险提醒 / 免责声明 → 反馈 → 历史 → 百宝箱 → 笔记 → 内测迭代。
