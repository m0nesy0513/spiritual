# ARCHITECTURE\.md

## **文档目的**

本文档用于将《社交媒体焦虑分析工具 PRD（十次修订版）》中已经确认的产品需求，转换为开发可执行的技术架构说明。

本文档不替代 PRD，不修改 PRD，不重新审查 PRD，不删减、弱化或重新解释 PRD 中已经确认的产品需求。

当某些技术实现细节在 PRD 中未明确时，本文档会给出合理默认方案，并明确标记为 **「架构建议」**。

本文档仅描述技术架构，不包含具体代码实现，不展开完整数据库字段设计，不展开完整 API 文档，不生成任务拆解与测试用例。

## **产品技术目标**

本项目 V0\.1 是 **朋友内测完整体验版**，不是极简 MVP，也不是粗糙 Demo。技术架构需要支撑完整的产品闭环：

登录 / 注册 → 首次进入引导 → 首页 → 上传截图 → OCR / 手动补充文字 → AI 分析 → 知识库匹配 → 分析结果 → 高风险提醒 → 反馈 → 历史记录 → 百宝箱 → 个人笔记 → 我的页面 → 管理员后台公共内容管理

## **总体架构概览**

本项目采用 **前后端一体的 Next\.js 架构**，以网站形式交付，手机端优先，电脑端兼容。

整体链路如下：

```Plain Text
用户浏览器
  ↓
Next.js 前端页面
  ↓
服务端 API
  ↓
业务 Service Layer
  ↓
Database / File Storage / AI Provider / OCR Provider
  ↓
管理员后台公共内容管理
```

更细化的文本架构图：

```Plain Text
Browser / Mobile Web
  ↓
Next.js App Router
  ├─ Public Pages
  ├─ Auth Pages
  ├─ User Pages
  ├─ Analysis Pages
  ├─ Knowledge Pages
  ├─ Note Pages
  └─ Admin Pages
  ↓
API Layer
  ├─ Auth API
  ├─ User API
  ├─ Upload API
  ├─ OCR API
  ├─ Analysis API
  ├─ History API
  ├─ Feedback API
  ├─ Knowledge API
  ├─ Note API
  ├─ Search API
  └─ Admin API
  ↓
Service Layer
  ├─ AuthService
  ├─ UserService
  ├─ UploadService
  ├─ OCRService
  ├─ AnalysisService
  ├─ KnowledgeService
  ├─ RiskService
  ├─ HistoryService
  ├─ FeedbackService
  ├─ NoteService
  ├─ SearchService
  └─ AdminService
  ↓
Infrastructure Layer
  ├─ MariaDB / MySQL
  ├─ Local File Storage or Object Storage
  ├─ DeepSeek API（多模态，OCR 由 AI 多模态能力一并处理）
  ├─ Alibaba Cloud SMS（阿里云短信）
  ├─ Resend（邮件服务）
  └─ 验证码：内测阶段不启用，仅保留接口与 UI
  ↓
Deployment
  ├─ Alibaba Cloud ECS
  ├─ Node.js Runtime
  ├─ PM2
  ├─ Nginx Reverse Proxy
  ├─ HTTPS Certificate
  └─ Backup / Logs
```

架构核心原则：

1. **PRD 优先**：技术实现不能反向修改产品需求。

2. **账号隔离优先**：所有私人数据必须按用户账号隔离。

3. **AI 服务层封装**：前端不直接调用 AI，AI 不直接裸跑。

4. **知识库支撑 AI**：AI 分析前必须经过知识库匹配流程。

5. **管理员边界明确**：管理员只能管理公共内容和用户主动提交的提议 / 反馈。

6. **手机端优先**：核心流程必须适配移动端浏览器。

7. **国内可用优先**：不引入国外强依赖服务，优先选择可自部署、可控、国内网络可用方案。

## **推荐技术栈**

不推荐第一版引入：

- Supabase 等外部强依赖平台；

- 国外云函数 / 国外对象存储作为核心链路；

- 复杂微服务；

- Kubernetes；

- 独立向量数据库作为强依赖；

- 大型消息队列；

- 复杂实时同步系统。

## **后端模块架构**

后端采用 API Layer \+ Service Layer \+ Repository Layer 的分层方式。

API Layer 负责：

- 接收请求；

- 登录态解析；

- 参数校验；

- 调用对应 Service；

- 返回统一响应；

- 不写复杂业务逻辑。

Service Layer 负责：

- 执行业务规则；

- 调用数据库、AI、OCR、文件存储；

- 处理事务；

- 执行权限校验；

- 生成业务状态。

Repository Layer 负责：

- 封装数据库读写；

- 不处理业务规则；

- 不直接暴露给前端。

### **7\.1 AuthService**

**负责：**

- 注册；

- 登录；

- 退出登录；

- 找回 / 重置密码；

- 修改密码；

- 手机号 / 邮箱验证码发送与校验；

- 协议、隐私政策、免责声明确认状态记录；

- 注册成功后自动登录；

- 登录后根据用户状态跳转首次进入引导或首页。

**不能做：**

- 不能读取用户分析历史内容；

- 不能处理管理员公共内容；

- 不能直接处理 AI 分析；

- 不能绕过协议与免责声明确认。

**交互模块：**

- UserService：创建用户、读取用户状态；

- VerificationService：验证码生成、校验、冷却；

- SessionService：登录态创建与销毁；

- Mail / SMS Adapter：发送验证码。

### **7\.2 UserService**

**负责：**

- 用户资料读取与修改；

- 用户名修改；

- 头像上传与替换；

- 绑定第二个凭证；

- 参考历史默认偏好设置；

- 退出登录前状态处理；

- 注销账号；

- 注销时物理删除私人数据；

- 注销时用户提议 / 反馈匿名化处理。

**不能做：**

- 不能修改 AI 分析原文；

- 不能删除公共知识库内容；

- 不能让普通用户变成管理员；

- 不能查看其他用户私人数据。

**交互模块：**

- AuthService：账号凭证；

- UploadService：头像文件；

- HistoryService：注销时删除历史；

- NoteService：注销时删除笔记；

- FeedbackService：匿名化用户提议 / 反馈；

- FileStorage：删除头像及私人附件。

### **7\.3 UploadService**

**负责：**

- 单张截图上传；

- 图片格式校验；

- 图片大小校验；

- 临时文件保存；

- 图片预览所需元信息；

- 重新上传覆盖当前临时图；

- 用户主动保存原截图时将临时文件转为持久文件；

- 历史删除或注销时删除附件。

**不能做：**

- 不能直接调用 AI 生成分析；

- 不能保存超出 PRD 支持范围的多图、视频、PDF；

- 不能默认永久保存用户原截图；

- 不能绕过文件权限控制。

**交互模块：**

- AnalysisService：提供分析所需图片文件；

- OCRService：提供 OCR 输入；

- HistoryService：绑定持久附件到历史记录；

- FileStorage：本地或对象存储读写。

### **7\.4 OCRService**

**负责：**

- 对上传截图进行文字识别（由 DeepSeek 多模态能力一并处理，不单独接入第三方 OCR 服务）；

- 生成完整识别文字；

- 判断 OCR 是否识别困难；

- 为图片预览弹层提供是否需要手动补充关键文字的判断；

- 保存用户手动补充关键文字；

- 将 OCR 文本作为知识库检索和 AI 辅助输入。

**不能做：**

- 不能替代 AI 多模态看图分析；

- 不能把 OCR 结果当作唯一事实来源；

- 不能在用户未选择保存完整识别文字时永久保存完整 OCR 文本；

- 不能将用户手动补充关键文字与完整 OCR 文本混为同一字段。

**交互模块：**

- UploadService：读取图片；

- AnalysisService：提供 OCR 辅助文本；

- KnowledgeService：提供检索关键词；

- HistoryService：按用户选择保存识别文字或补充文字。

### **7\.5 AnalysisService**

**负责：**

- 分析主流程编排；

- 接收图片、用户感受、焦虑强度、来源平台、参考历史开关；

- 调用 OCRService；

- 调用 KnowledgeService 匹配知识条目；

- 组织 AI 输入上下文；

- 调用 AI Provider；

- 解析 AI 结构化输出；

- 生成系统渲染模块所需数据；

- 创建或更新历史记录；

- 执行重新分析规则；

- 触发 RiskService 判断高风险提醒状态。

**不能做：**

- 不能跳过知识库匹配直接裸跑 AI；

- 不能让 AI 生成免责声明；

- 不能让 AI 直接决定百宝箱推荐最终展示；

- 不能伪造来源 / 参考依据；

- 不能覆盖不该覆盖的历史记录；

- 不能在用户无权限时读取历史记录。

**交互模块：**

- UploadService：获取图片；

- OCRService：获取 OCR 文本和补充文字；

- KnowledgeService：匹配知识条目；

- RiskService：识别与保存高风险状态；

- HistoryService：创建 / 更新历史；

- AI Adapter：调用大模型；

- SearchService：间接依赖知识标签和历史摘要。

### **7\.6 KnowledgeService**

**负责：**

- 知识库条目查询；

- 已启用知识条目筛选；

- 分类与标签管理；

- 知识条目匹配；

- 百宝箱列表与详情；

- 百宝箱相关推荐；

- 结果页相关百宝箱推荐；

- 来源 / 参考依据快照生成；

- 0 匹配知识条目时返回特殊提示。

**不能做：**

- 不能引用停用知识条目进入 AI 上下文；

- 不能在 0 匹配时伪造来源；

- 不能读取用户私人历史或笔记正文；

- 不能将百宝箱推荐与来源 / 参考依据混淆。

**交互模块：**

- AnalysisService：提供知识条目注入；

- AdminService：维护知识条目；

- SearchService：百宝箱搜索；

- HistoryService：保存来源快照。

### **7\.7 HistoryService**

**负责：**

- 分析成功后自动创建历史；

- 历史列表；

- 历史详情；

- 历史筛选；

- 历史搜索；

- 历史删除；

- 历史详情中的备注和标签；

- 重新分析入口的数据恢复；

- 来源 / 参考依据快照；

- 免责声明快照；

- 高风险标记与提醒状态保存；

- 删除历史时解除个人笔记关联。

**不能做：**

- 不能允许用户访问非本人历史 ID；

- 不能删除历史时顺带删除用户个人笔记；

- 不能在历史详情中实时替换旧来源快照；

- 不能把历史恢复进入结果页的重新分析当成覆盖旧记录。

**交互模块：**

- AnalysisService：创建 / 更新历史；

- FeedbackService：更新反馈；

- NoteService：解除关联；

- RiskService：读取提醒状态；

- UploadService / FileStorage：删除附件。

### **7\.8 FeedbackService**

**负责：**

- 分析反馈提交；

- 反馈页完成状态；

- 反馈提交失败保留输入；

- 用户提议 / 反馈提交；

- 管理员查看用户主动提交的提议 / 反馈；

- 用户注销时匿名化用户提议 / 反馈。

**不能做：**

- 不能修改 AI 原始分析内容；

- 不能让用户在历史详情页补交本次分析反馈；

- 不能让管理员通过反馈后台查看私人分析记录；

- 不能将产品内分析反馈和用户提议 / 反馈混成同一业务含义。

**交互模块：**

- HistoryService：更新分析记录反馈字段；

- AdminService：查看和处理用户提议；

- UserService：注销匿名化。

### **7\.9 NoteService**

**负责：**

- 个人笔记新增、编辑、删除；

- 笔记搜索；

- 笔记标签；

- 关联历史记录；

- 关联百宝箱内容；

- 关联来源快照；

- 历史删除后的关联解除；

- 百宝箱停用后的关联展示状态。

**不能做：**

- 不能覆盖 AI 结论；

- 不能访问其他用户笔记；

- 不能把历史标签和笔记标签混用为同一套标签系统；

- 不能在百宝箱停用后继续跳转到停用详情页。

**交互模块：**

- HistoryService：关联历史；

- KnowledgeService：关联百宝箱；

- SearchService：笔记搜索；

- UserService：注销时删除笔记。

### **7\.10 AdminService**

**负责：**

- 管理员身份校验；

- 百宝箱管理；

- 百宝箱首页推荐；

- 首页名言管理；

- 好歌推荐管理；

- 轻柔情绪问候维护；

- Coming Soon 文案维护；

- 产品说明维护；

- 免责声明维护；

- 新手教程内容维护；

- 用户提议 / 反馈处理状态修改；

- 系统文案设置。

**不能做：**

- 不能查看用户私人历史；

- 不能查看用户截图；

- 不能查看用户感受；

- 不能查看 AI 分析结果；

- 不能查看用户个人笔记；

- 不能删除免责声明核心条款；

- 不能让普通用户自助申请管理员。

**交互模块：**

- KnowledgeService：维护公共知识内容；

- FeedbackService：查看用户主动提交的提议 / 反馈；

- UserService：读取管理员身份；

- AuthService：管理员登录态；

- Config：管理员账号初始化配置。

### **7\.11 RiskService**

**负责：**

- 高风险信号识别结果接收；

- 根据 AI 输出、用户感受、焦虑强度判断是否触发高风险提醒；

- 保存高风险标记；

- 保存提醒状态：未触发 / 待处理 / 已处理；

- 控制结果页或历史详情页是否弹出全屏提醒；

- 用户点击继续查看或返回首页后更新为已处理。

**不能做：**

- 不能提供医疗诊断；

- 不能提供危机干预；

- 不能删除普通分析结果；

- 不能阻止分析结果和历史记录生成；

- 不能在已处理后反复弹出。

**交互模块：**

- AnalysisService：分析完成后触发；

- HistoryService：保存风险状态；

- Frontend Result Page：决定是否展示全屏弹层。

### **7\.12 SearchService**

**负责：**

- 首页统一搜索；

- 历史局部搜索；

- 百宝箱搜索；

- 个人笔记搜索；

- 搜索与筛选交集；

- 首页“查看更多”跳转到对应模块并携带关键词；

- 搜索无结果状态。

**不能做：**

- 不能跨用户搜索私人数据；

- 不能让未登录用户搜索首页、历史、百宝箱、笔记；

- 不能把 AI 关键词、历史自定义标签、百宝箱标签混淆；

- 不能绕过模块权限。

**交互模块：**

- HistoryService：历史搜索；

- KnowledgeService：百宝箱搜索；

- NoteService：笔记搜索；

- Auth / Policy：用户数据隔离。

## **核心业务流程架构**

### **8\.1 注册 / 登录流程**

#### **8\.1\.1 注册流程**

```Plain Text
用户进入注册页
  ↓
选择手机号或邮箱
  ↓
输入验证码接收凭证
  ↓
系统发送短信验证码或邮件验证码
  ↓
用户输入验证码、密码、用户名
  ↓
用户勾选：
  1. 同意用户协议和隐私政策
  2. 确认已阅读免责声明
  ↓
AuthService 校验
  ├─ 凭证格式
  ├─ 验证码是否正确
  ├─ 验证码是否过期
  ├─ 密码规则
  ├─ 用户名规则
  ├─ 协议与免责声明确认
  └─ 凭证是否已注册
  ↓
创建用户与登录凭证
  ↓
记录协议 / 隐私政策 / 免责声明确认状态
  ↓
注册成功后自动登录
  ↓
跳转首次进入引导页
```

注册规则：

- 手机号注册使用短信验证码；

- 邮箱注册使用邮件验证码；

- 验证码有效期 5 分钟；

- 验证码重发冷却 60 秒；

- 用户名必填，最多 20 字；

- 密码至少 8 位，包含字母和数字；

- 注册时必须确认用户协议、隐私政策、免责声明；

- 注册成功后自动登录；

- 新用户进入首次进入引导页。

#### **8\.1\.2 登录流程**

```Plain Text
用户进入登录页
  ↓
输入手机号 / 邮箱 + 密码
  ↓
AuthService 校验凭证
  ↓
校验成功
  ↓
创建登录态
  ↓
判断是否完成首次进入引导
  ├─ 未完成 → 跳转首次进入引导页
  └─ 已完成 → 跳转首页
```

登录失败状态：

- 账号不存在；

- 密码错误；

- 输入格式错误；

- 登录失败；

- 网络异常。

#### **8\.1\.3 找回 / 重置密码流程**

```Plain Text
用户进入找回 / 重置密码页
  ↓
第一步：输入已绑定手机号或邮箱
  ↓
发送短信 / 邮件验证码
  ↓
用户输入验证码
  ↓
验证码校验通过
  ↓
第二步：设置新密码
  ↓
AuthService 校验密码规则
  ↓
更新密码
  ↓
清除找回流程状态
  ↓
跳转登录页
  ↓
提示用户使用新密码重新登录
```

找回限制：

- 仅支持通过已绑定手机号或邮箱找回；

- 若唯一凭证不可用，第一版不提供人工找回；

- 重置成功后跳转登录页，不自动登录。

#### **8\.1\.4 首次进入引导流程**

```Plain Text
用户登录成功
  ↓
系统判断 onboarding_completed
  ↓
若未完成
  ↓
展示首次进入引导页
  ↓
用户点击继续或跳过
  ↓
标记该账号已处理首次引导
  ↓
进入首页
  ↓
第一次进入首页后展示新手教程遮罩
```

注意：

- 跳过引导不等于跳过免责声明；

- 免责声明已在注册时确认；

- 首次引导按账号维度记录；

- 换设备登录不重复自动展示。

### **8\.2 上传截图与分析流程**

```Plain Text
用户点击底部中间相机按钮
  ↓
进入上传截图页
  ↓
选择或拖拽单张图片
  ↓
UploadService 校验：
  ├─ 数量：仅 1 张
  ├─ 格式：PNG / JPG / JPEG / WEBP
  ├─ 大小：≤ 10MB
  └─ 类型：不支持 GIF / 视频 / PDF
  ↓
图片保存到临时目录
  ↓
页面展示缩略图
  ↓
用户可点击缩略图进入图片预览弹层
  ↓
OCRService 尝试识别文字
  ↓
若 OCR 识别困难
  ↓
提示重新上传或手动补充关键文字
  ↓
用户可填写：
  ├─ 感受，最多 500 字
  ├─ 焦虑强度，0-10 整数
  ├─ 截图来源平台
  ├─ 手动补充关键文字，最多 200 字
  └─ 是否参考历史摘要
  ↓
用户点击开始分析
  ↓
创建分析任务上下文
  ↓
进入分析等待页
  ↓
AnalysisService 执行分析
  ↓
分析成功
  ↓
创建历史记录
  ↓
跳转分析结果页
```

上传页离开保护：

- 用户上传截图或填写感受后离开上传页，需要提示当前内容会丢失；

- 站内跳转、底部导航、返回按钮需触发确认；

- 浏览器关闭 / 刷新第一版可不保证。

分析取消：

- 用户在等待页点击取消分析；

- 不保存分析结果；

- 返回上传页；

- 保留截图、感受、焦虑强度、来源平台、手动补充文字；

- 清除分析等待状态。

分析失败：

- 保留已上传截图、感受、焦虑强度、来源平台、手动补充关键文字；

- 提供重新分析、重新上传、返回首页。

### **8\.3 AI 分析流程**

```Plain Text
AnalysisService 接收分析请求
  ↓
读取临时截图
  ↓
调用 OCRService 获取 OCR 文字
  ↓
合并用户输入：
  ├─ 用户感受
  ├─ 焦虑强度
  ├─ 来源平台
  ├─ 手动补充关键文字
  └─ 是否参考历史摘要
  ↓
若用户开启参考历史
  ↓
读取历史摘要：
  ├─ 历史一句话摘要
  ├─ 人设类型
  ├─ 焦虑类型
  ├─ 关键词
  └─ 反馈概况
  ↓
KnowledgeService 匹配已启用知识条目
  ↓
按规则选择 5-8 条注入 AI
  ↓
若不足 5 条，注入全部
  ↓
若 0 条，允许基于通用知识生成，但来源区标注未匹配
  ↓
组装 AI 输入上下文
  ↓
调用多模态 AI Provider
  ↓
AI 优先直接看图
  ↓
OCR 文字和用户输入作为辅助
  ↓
AI 返回结构化结果
  ↓
AnalysisService 校验输出结构
  ↓
KnowledgeService 生成相关百宝箱推荐
  ↓
RiskService 判断高风险状态
  ↓
HistoryService 保存历史记录、来源快照、免责声明快照
  ↓
返回结果页数据
```

AI 输入包括：

- 用户上传的截图原图；

- OCR 识别文字；

- 用户手动补充关键文字；

- 用户感受；

- 分析前焦虑强度；

- 截图来源平台；

- 相关已启用知识条目；

- 用户主动开启时的历史摘要。

AI 输出结构包括 11 个 AI 生成模块：

1. 一句话摘要；

2. 关键词；

3. 截图内容概括；

4. 引起焦虑的人设 / 人格化形象类型；

5. 焦虑自身类型；

6. 包装与隐藏信息拆解；

7. 比较陷阱分析；

8. 为什么你会焦虑；

9. CBT / 心理学方法辅助；

10. 具体建议；

11. 来源 / 参考依据。

系统渲染模块包括：

1. 相关百宝箱推荐；

2. 免责声明。

AI 输出约束：

- 一句话摘要最多 30 字；

- 关键词 1\-8 个；

- 截图内容概括最多 100 字；

- 具体建议 1\-5 条，每条不超过 100 字；

- 信息不足时显示“当前信息不足，无法判断”；

- 不得省略必需模块；

- 不得生成免责声明；

- 不得伪造来源；

- 不得提供医疗诊断或心理治疗。

0 匹配知识条目处理：

```Plain Text
KnowledgeService 匹配结果 = 0
  ↓
AnalysisService 仍允许 AI 基于通用知识生成
  ↓
来源 / 参考依据显示：
“本次分析未匹配到特定知识条目，基于通用知识生成”
  ↓
不得伪造知识条目标题、分类或来源
```

### **8\.4 高风险提醒流程**

```Plain Text
AI 分析完成
  ↓
RiskService 检查高风险信号
  ↓
判断依据：
  ├─ 明确自伤 / 自杀 / 伤害自己表达
  ├─ 不想活 / 想消失 / 想离开世界
  ├─ 极端绝望
  ├─ 危险行动暗示
  └─ 焦虑强度 ≥ 8 且同时出现明显危险表达
  ↓
若未触发
  ↓
历史记录 risk_state = 未触发
  ↓
正常展示结果页
```

触发时：

```Plain Text
AI 识别高风险
  ↓
系统先生成完整分析结果
  ↓
系统先创建历史记录
  ↓
保存高风险标记
  ↓
提醒状态 = 待处理
  ↓
跳转结果页
  ↓
弹出全屏高风险提醒遮挡结果
  ↓
用户必须选择：
  ├─ 我已了解，继续查看分析
  └─ 返回首页
  ↓
提醒状态 = 已处理
```

异常退出时：

```Plain Text
高风险提醒状态 = 待处理
  ↓
用户关闭浏览器或异常退出
  ↓
再次打开该历史详情
  ↓
仍触发一次全屏提醒
  ↓
用户选择继续查看或返回首页
  ↓
提醒状态 = 已处理
  ↓
后续不再重复弹出
```

架构要求：

- 高风险提醒不能阻止历史记录保存；

- 高风险提醒不能删除普通分析结果；

- 高风险提醒不是医疗诊断；

- 高风险标记随历史记录删除而删除；

- 用户选择返回首页后，结果和历史仍保留。

### **8\.5 反馈与历史保存流程**

```Plain Text
AI 分析成功
  ↓
HistoryService 自动创建历史记录
  ↓
保存：
  ├─ 用户感受
  ├─ 分析前焦虑强度
  ├─ 截图来源平台
  ├─ 手动补充关键文字
  ├─ AI 结构化分析结果
  ├─ 一句话摘要
  ├─ 关键词
  ├─ 人设类型
  ├─ 焦虑类型
  ├─ 历史摘要
  ├─ 分析时间
  ├─ 来源 / 参考依据快照
  ├─ 免责声明快照
  └─ 高风险状态
  ↓
用户进入结果页
  ↓
用户可进入反馈页
```

反馈提交：

```Plain Text
用户填写反馈
  ↓
FeedbackService 校验
  ↓
更新当前历史记录
  ↓
反馈页进入完成状态
  ↓
展示：
  ├─ 回首页
  ├─ 看百宝箱
  └─ 查看历史
```

反馈可跳过：

- 用户跳过反馈不影响历史保存；

- 历史记录保留，反馈字段为空；

- 历史列表显示“未反馈”；

- 第一版不允许在历史详情页补充本次分析反馈。

历史详情展示快照：

- 来源 / 参考依据使用分析时快照；

- 免责声明使用分析时快照；

- 后续知识库编辑、停用不影响历史详情；

- 后续免责声明修改不影响历史详情。

### **8\.6 重新分析流程**

重新分析分三种场景。

#### **8\.6\.1 上传页首次结果页未反馈前重新分析**

```Plain Text
用户从上传页完成首次分析
  ↓
进入结果页
  ↓
尚未提交反馈
  ↓
点击重新分析
  ↓
复用当前输入
  ↓
重新触发 AI 分析
  ↓
覆盖当前历史记录的分析结果内容
  ↓
保留原历史记录 ID
  ↓
保留原分析时间
  ↓
保留用户此前对保存原截图 / 保存完整识别文字的选择
```

注意：

- 不删除重建历史记录；

- 不生成新历史记录；

- 只更新当前历史记录中的分析结果内容。

#### **8\.6\.2 从历史详情页重新分析**

```Plain Text
用户打开历史详情页
  ↓
点击“基于这条记录重新分析”
  ↓
默认复用：
  ├─ 原截图
  ├─ 原感受
  ├─ 原焦虑强度
  ├─ 原截图来源平台
  └─ 原参考历史设置
  ↓
重新触发 AI 分析
  ↓
生成新的历史记录
  ↓
不覆盖旧记录
```

#### **8\.6\.3 从历史恢复进入结果页后重新分析**

```Plain Text
用户通过历史记录 ID 恢复进入结果页
  ↓
点击重新分析
  ↓
按钮文案为“基于这条记录重新分析”
  ↓
生成新的历史记录
  ↓
不覆盖旧记录
```

适用规则：

- 已提交反馈的记录重新分析，也生成新历史记录；

- 从历史详情页或历史恢复进入结果页，都不覆盖旧记录；

- 第一版不限制重新分析次数；

- 页面可提示“重新分析会从同一材料出发，结果可能相似”。

```Plain Text
  ↓
用户二次确认
  ↓
UserService 执行注销
  ↓
物理删除私人数据：
  ├─ 历史记录

  ├─ 感受

  ├─ 反馈记录
  ├─ 后续感受
  ├─ 备注
  ├─ 个人笔记

  ├─ 头像

  ├─ 账号凭证
  └─ 注册时间等账号关联数据

用户提议 / 反馈匿名化处理
```

用户提议 / 反馈匿名化保留规则：

- 删除用户名；

- 删除联系方式；

- 保留提议类型；

- 保留处理状态；

## **文件与图片存储架构**

### **10\.1 截图上传临时处理**

```Plain Text
用户上传截图
  ↓
UploadService 校验格式和大小
  ↓
保存到 uploads/tmp 或对象存储临时区
  ↓
生成临时 file token
  ↓
供 OCR 和 AI 分析使用
  ↓
分析完成后根据用户选择决定是否持久保存
```

临时处理规则：

- 仅支持单张截图；

- 临时文件需要绑定当前登录用户；

- 临时文件不能被其他用户访问；

- 临时文件应设置过期清理机制；

- 分析失败或取消时保留当前会话所需文件，便于重新分析；

- 用户返回首页或超时后可由清理任务删除。

### **10\.2 用户默认不保存原截图**

PRD 明确：结果页中“保存原截图”和“保存完整识别文字”默认不勾选。

架构处理：

```Plain Text
分析需要临时使用截图
  ↓
AI 分析完成
  ↓
用户未勾选保存原截图
  ↓
历史记录不关联原截图持久文件
  ↓
临时文件进入待清理状态
  ↓
历史列表展示统一占位图
```

注意：

- 不保存原截图不影响 AI 分析；

- 不保存原截图不影响历史记录中的分析结果；

- 历史详情展示“未保存截图”。

### **10\.3 用户勾选保存后的存储**

```Plain Text
用户在结果页勾选保存原截图
  ↓
UploadService 将临时文件迁移到持久目录
  ↓
FileAsset 创建持久文件记录
  ↓
HistoryService 将 FileAsset 关联到 AnalysisRecord
```

建议文件路径结构：

```Plain Text
uploads/
  analysis/
    user-{userId}/
      record-{recordId}/
        original.webp
        ocr-text.txt
```

**架构建议**：

- 原图可保留原格式；

- 如后续需要节省空间，可转码为 WEBP；

- 第一版不建议自动压缩原截图，避免影响用户保存预期；

- 文件名不使用用户原始文件名，避免泄露信息。

### **10\.4 完整识别文字存储**

规则：

- 完整 OCR 识别文字默认不保存；

- 用户勾选保存完整识别文字后，才持久保存；

- 手动补充关键文字与完整识别文字分开保存；

- 即使用户未保存完整识别文字，只要用户曾手动补充关键文字，历史详情仍展示该补充文字。

### **10\.5 历史删除时同步删除附件**

```Plain Text
用户删除历史
  ↓
HistoryService 查询关联 FileAsset
  ↓
FileStorage 删除物理文件
  ↓
删除 FileAsset 记录
  ↓
删除 AnalysisRecord 相关数据
```

要求：

- 数据库删除与文件删除需要有失败补偿机制；

- 若文件删除失败，需要记录日志；

- 不允许删除其他用户文件；

- 删除历史不删除个人笔记，只解除关联。

### **10\.6 头像存储**

头像规则：

- 支持 JPG / PNG；

- 大小上限 2MB；

- 超过 2MB 提示，不自动压缩；

- 未上传头像时，使用系统根据用户名首字母生成的彩色默认头像。

头像路径建议：

```Plain Text
uploads/
  avatars/
    user-{userId}/
      avatar.png
```

### **10\.7 本地存储与对象存储对比**

### **10\.8 第一版推荐方案**

**架构建议：第一版优先使用本地服务器存储。**

理由：

1. 朋友内测规模可控；

2. 文件保存默认关闭，真实持久文件量不会很大；

3. 部署和调试简单；

4. 数据更集中，权限更容易控制；

5. 避免第一版引入对象存储配置复杂度。

同时需要预留 Storage Adapter：

```Plain Text
StorageService
  ├─ LocalStorageAdapter
  └─ ObjectStorageAdapter
```

后续切换对象存储时，不应影响上层业务服务。

## **AI 与知识库架构**

### **11\.1 AI 调用原则**

AI 不直接裸跑，必须经过服务层封装。

```Plain Text
前端
  ↓
Analysis API
  ↓
AnalysisService
  ↓
KnowledgeService / OCRService / RiskService
  ↓
AI Adapter
  ↓
AI Provider
```

前端不能直接持有 AI API Key。

AI Provider 的请求参数、模型名称、超时时间、重试策略由服务端统一管理。

### **11\.2 AI Provider 封装**

AI 调用需要封装为统一接口：

**架构建议**：AI Adapter 设计成可切换 Provider，便于在 DeepSeek 或其他国内可用多模态模型之间切换。

### **11\.3 知识库匹配流程**

```Plain Text
AnalysisService 提取检索信息
  ├─ OCR 关键词
  ├─ 用户感受关键词
  ├─ 截图来源平台
  ├─ 人设类型候选
  ├─ 焦虑类型候选
  └─ 手动补充关键文字
  ↓
KnowledgeService 查询已启用知识条目
  ↓
按分类、标签、正文、适用场景匹配
  ↓
排序
  ↓
选择 5-8 条
  ↓
注入 AI 上下文
```

匹配规则：

- 优先已启用知识条目；

- 优先适用场景与本次分析最匹配的条目；

- 每次注入 5\-8 条；

- 不足 5 条时注入全部；

- 0 条时允许 AI 基于通用知识生成；

- 0 条时不得伪造来源。

### **11\.4 知识库最小内容包**

架构必须支持至少 60 条知识内容，并支持以下核心分类：

- 焦虑类型解释；

- 人设图鉴；

- 社交媒体套路拆解；

- CBT / 心理学方法；

- 情绪急救步骤；

- 行动建议库；

- 自我安抚 / 反比较训练；

- 典型案例。

完整内测最低条件：

- 总计不少于 60 条；

- 每个核心分类至少 3 条已启用内容；

- 无空标题；

- 无空正文。

### **11\.5 相关百宝箱推荐**

相关百宝箱推荐由系统生成，不属于 AI 直接输出。

```Plain Text
AI 分析完成
  ↓
AnalysisService 获取本次关键词、人设类型、焦虑类型
  ↓
KnowledgeService 根据匹配知识条目的标签和分类排序
  ↓
生成 0-3 条推荐卡片
  ↓
结果页展示
```

规则：

- 优先展示 3 条；

- 不足 3 条展示实际数量；

- 无匹配时显示“暂无相关推荐”或“当前无匹配知识条目”；

- 点击进入百宝箱详情页；

- 不与来源 / 参考依据混淆。

### **11\.6 来源 / 参考依据**

来源 / 参考依据属于 AI 输出模块，但引用范围受系统控制。

来源展示内容包括：

- 知识条目标题；

- 分类；

- 简短摘要；

- 为什么用于本次分析。

历史记录保存来源快照：

```Plain Text
分析时引用知识条目
  ↓
生成来源说明
  ↓
保存标题、分类、摘要、关联说明
  ↓
历史详情读取快照
  ↓
不受后续知识库编辑 / 停用影响
```

0 匹配时：

```Plain Text
本次分析未匹配到特定知识条目，基于通用知识生成
```

### **11\.7 免责声明**

免责声明由系统渲染，不属于 AI 输出。

规则：

- 结果页每次显示；

- 默认展开；

- 不可折叠；

- 来源于管理员后台维护的全局免责声明；

- 核心条款不可删除；

- 历史记录保存免责声明快照。

```Plain Text

  ↓

  ↓

  ↓
```

- 后端 API 必须防；

- 数据库查询必须带 user\_id；

- 不允许通过 ID 枚举获取他人记录。

### **12\.4 管理员路由守卫**

管理员页面访问流程：

```Plain Text
用户访问 /admin
  ↓
检查是否登录
  ↓
检查是否管理员
  ↓
非管理员 → 403
  ↓
管理员 → 进入后台
```

管理员身份指定：

- 由项目创建者预先指定；

- 第一版通过部署配置或初始化数据指定；

- 不采用第一个注册账号自动成为管理员；

- 普通用户不能申请管理员；

- 更换管理员需由项目创建者修改配置或初始化数据。

### **12\.5 管理员 API 权限校验**

所有 `/api/admin/*` 接口必须执行管理员校验。

不能只依赖前端隐藏入口。

普通用户直接调用管理员 API 时必须返回 403。

### **12\.6 管理员不能查看用户私人内容**

管理员不能查看：

- 用户历史；

- 用户截图；

- 用户感受；

- 用户分析结果；

- 用户反馈记录；

- 后续感受；

- 个人笔记；

- 标签；

- 备注。

管理员可以查看：

- 用户主动提交的提议 / 反馈；

- 用户在提议 / 反馈中主动填写的联系方式；

- 提议类型；

- 提交时间；

- 处理状态。

### **12\.7 用户提议 / 反馈与私人分析记录隔离**

用户提议 / 反馈是用户主动提交给管理员的公共处理对象。

分析反馈是某次 AI 分析后的私人记录，不应被管理员查看。

二者在架构上应拆分：

```Plain Text
AnalysisFeedback
  └─ 私人分析反馈，仅用户自己可见

UserSuggestion
  └─ 用户主动提交给管理员的产品反馈，管理员可处理
```

### **12\.8 高风险提醒不是医疗诊断**

高风险提醒必须明确：

- 产品不替代专业医疗；

- 不替代心理咨询；

- 不替代心理治疗；

- 不提供医学诊断；

- 不提供危机干预；

- 严重风险时建议寻求专业帮助。

### **12\.9 文件访问权限控制**

文件访问必须经过权限校验。

推荐方式：

```Plain Text
用户请求图片
  ↓
通过受控 API 访问
  ↓
校验当前用户是否拥有该 FileAsset
  ↓
通过后返回文件流或临时访问地址
```

不推荐直接暴露可猜测静态路径。

如使用本地存储：

- 上传目录不要放在公开静态目录；

- 通过 API 读取；

- 文件名使用随机 ID；

- 不使用原始文件名。

如使用对象存储：

- 私有 Bucket；

- 通过服务端签名 URL 临时访问；

- URL 短期有效。

## **部署架构建议**

### **16\.1 服务器部署方式**

面向中国大陆网络环境，推荐使用国内云服务器。

**架构建议：**

- 阿里云 ECS；

- Alibaba Cloud Linux / Ubuntu LTS；

- 单机部署 Next\.js \+ MariaDB \+ Nginx；

- 第一版不拆多台服务器；

- 后续用户增长后再拆数据库和文件存储。

部署结构：

```Plain Text
Internet
  ↓
Nginx :443
  ↓
Next.js Node Server :3000
  ↓
MariaDB :3306
  ↓
Local File Storage
  ↓
AI / OCR / SMS / Email Provider
```

### **16\.2 Node\.js 运行**

- 使用 Node\.js LTS；

- Next\.js 构建后以 Node 服务运行；

- 服务仅监听本地端口；

- 外部访问统一经过 Nginx。

### **16\.3 PM2 管理**

PM2 负责：

- 启动 Next\.js 服务；

- 进程崩溃自动重启；

- 查看运行状态；

- 管理环境变量；

- 保存应用日志；

- 支持开机自启。

### **16\.4 Nginx 反向代理**

Nginx 负责：

- HTTPS 终止；

- HTTP 跳转 HTTPS；

- 反向代理到 Node\.js；

- 设置上传大小限制；

- 静态资源缓存；

- 隐藏后端真实端口；

- 基础安全 Header。

需要确保：

- 上传大小限制大于 10MB；

- 请求超时时间能覆盖 AI 分析等待；

- 不直接暴露用户上传目录。

### **16\.5 MariaDB 部署**

第一版可在同一 ECS 上部署 MariaDB。

要求：

- 数据库仅允许本机访问；

- 使用独立数据库用户；

- 不使用 root 账号给应用连接；

- 定期备份；

- 使用 utf8mb4；

- 表引擎使用 InnoDB；

- 关键外键关系或业务删除逻辑需要明确。

### **16\.6 环境变量管理**

- 不把密钥写进代码；

- `.env` 文件不提交仓库；

- 生产环境使用服务器环境变量或 PM2 环境配置；

- 区分开发、测试、生产环境；

- AI Key、JWT Secret、数据库密码、短信服务密钥必须保密。

### **16\.7 日志管理**

日志类型：

注意：

- 不记录用户完整私人内容；

- 不记录完整截图内容；

- 不在日志中输出 API Key；

- 对 AI 输入输出日志应做脱敏或只记录摘要信息。

### **16\.8 文件目录权限**

- 上传目录由应用运行用户读写；

- 不给 Nginx 直接公开访问私人文件目录；

- 临时目录和持久目录分离；

- 定期清理临时文件；

- 删除历史和注销账号时同步删除文件。

### **16\.9 HTTPS**

必须使用 HTTPS。

**架构建议：**

- 使用国内云服务商证书；

- 或使用可用的免费证书方案；

- Nginx 负责证书配置；

- Cookie 设置 Secure、HttpOnly、SameSite。

### **16\.10 备份策略**

第一版至少需要：

- 数据库每日备份；

- 上传文件目录定期备份；

- 备份文件与生产数据分离存放；

- 保留最近若干天备份；

- 定期验证恢复流程。

**架构建议：**

- 朋友内测阶段可使用脚本备份；

- 后续可迁移到对象存储或云备份服务；

- 备份中包含用户私人数据，需要严格控制访问权限。

## **架构结论**

本架构支持《社交媒体焦虑分析工具 PRD（十次修订版）》定义的 **V0\.1 朋友内测完整体验版**。

本架构的核心优先级是：

1. 账号安全；

2. 协议、隐私政策与免责声明合规确认；

3. 单张截图上传与 AI 分析闭环；

4. AI 输出结构一致性；

5. 知识库对 AI 分析的支撑；

6. 高风险提醒三态处理；

7. 历史记录自动保存与快照机制；

8. 用户数据隔离与权限控制；

9. 管理员只能管理公共内容；

10. 手机端核心流程可用。

后续应继续生成：

- `DATABASE.md`：完整数据库表结构、字段、索引、关系、删除规则；

- `API.md`：完整接口路径、请求参数、响应结构、错误码；

- `TASKS.md`：开发任务拆解与优先级；

- `TEST_CASES.md`：核心流程、权限、错误状态、移动端、AI 输出、管理员后台测试用例。

本文档作为技术架构基准，后续技术文档不得反向修改 PRD 中已经确认的产品需求。
